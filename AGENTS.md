-- Visual snapshot testing (for detecting UI regressions visually):
   Uses Playwright + headless Chromium to capture screenshots of the Flutter web app
   in predefined viewport/state scenarios, then pixel-diffs against reference baselines.
   ── Scenarios: 10 total (7 desktop, 1 tablet, 2 mobile) — defined in scripts/snapshots.mjs
   ── Test TLE cache: scripts/generate-test-tle.py creates 191 realistic orbital objects
      in build/web/api/tle.json so the app doesn't hang on CORS fallback during capture
   ── Server: scripts/serve_spa.py is a Python SPA-compatible HTTP server
   ── Diffing: scripts/visual-diff.mjs uses pixelmatch for pixel-perfect comparison
   ── Workflow:
       npm run snapshots:build     # flutter build web --wasm + capture all scenarios
       npm run visual:diff         # compare current vs reference (exit 1 if mismatches)
       npm run visual:update       # accept current as new baseline
       npm run visual:report       # open snapshots/report.html
   ── Reference baselines live in snapshots/reference/ (committed to git)
   ── Current/diff outputs in snapshots/current/ and snapshots/diff/ (gitignored)
   ── Dependencies: playwright, pixelmatch, pngjs (in package.json devDeps)
   ── First time setup: npm install && npx playwright install chromium
   ── After intentional UI changes, run `npm run visual:update` and commit baselines.

-- DigitalOcean droplet workflow (for one-off full TLE fetch when server IP is blocked):
  WARNING: Droplets cost money ($4-6/mo prorated). Destroy immediately after use.
  The DO API token is at /home/bem/.do_token (env var $DO_TOKEN).
  CelesTrak rate-limits per IP aggressively — a fresh DO IP can do ONE successful fetch
  before getting rate-limited too. Steps:
    1. Create ONE droplet with your local ed25519 key (already registered as DO SSH key)
    2. Wait for it, ssh in, copy fetch-tle.py, run it ONCE
    3. Generate an SSH key on the droplet, add its pubkey to prod's ~/.ssh/authorized_keys
    4. scp the result to root@server.4ft.me:/opt/junk/api/tle.json
    5. Destroy the droplet immediately
    6. Remove the temp SSH key from prod's authorized_keys
  ⚠ Do NOT run fetch-tle.py twice from the same IP — the second run will be rate-limited
    and overwrite the good cache with a partial one (~1,480 objects).
    If you mess up, destroy the droplet and start fresh — don't retry from the same IP.
